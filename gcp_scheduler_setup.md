# GCP Secure Architecture: Public Search API & Private Scheduler Sync

This guide details how to deploy your movie search and sync services to Google Cloud Run, limiting instances to save cost, and configuring Cloud Scheduler to securely trigger updates.

---

## Architecture Overview

We recommend deploying the **same container image** as **two separate Cloud Run services**. Because Cloud Run is billed purely by active request usage, running two services costs exactly the same as one, but provides clean security boundaries:

1. **`movie-search-api`** (Public):
   - Deployed with `--allow-unauthenticated` so web/app clients can query the search API.
   - Deployed with `--max-instances=1` to prevent cost spikes.
2. **`movie-sync-service`** (Private):
   - Deployed with `--no-allow-unauthenticated` to block all public access.
   - Deployed with `--max-instances=1` to ensure sync runs sequentially without race conditions.
   - Triggered securely by Cloud Scheduler using an **OIDC token** generated by a dedicated IAM service account.

```
                  ┌─────────────────┐
                  │ Web/Mobile App  │
                  └────────┬────────┘
                           │ (Public Search Request)
                           ▼
             ┌───────────────────────────┐
             │     movie-search-api      │ (Public Cloud Run, max-instances=1)
             └───────────────────────────┘
                           │
                           ▼
                  ┌─────────────────┐
                  │  GCP Firestore  │
                  └─────────────────┘
                           ▲

             ┌───────────────────────────┐
             │    movie-sync-service     │ (Private Cloud Run, max-instances=1)

```

## 1. Deploy the Services with `gcloud` CLI

Replace `<PROJECT_ID>` with your actual GCP project ID (e.g., `hami-review`).

### Deploy Service 1: Movie Search API (Public)
Deploy the public-facing search API, constraining it to 1 instance:
```bash
gcloud run deploy movie-search-api \
  --image gcr.io/<PROJECT_ID>/movie-search-api \
  --platform managed \
  --region asia-east1 \
  --max-instances=1 \
  --allow-unauthenticated
```

### Deploy Service 2: Movie Sync Service (Private)
Deploy the private sync service using the same image, but blocking public access and setting max instances to 1:
```bash
gcloud run deploy movie-sync-service \
  --image gcr.io/<PROJECT_ID>/movie-search-api \
  --platform managed \
  --region asia-east1 \
  --max-instances=1 \
  --no-allow-unauthenticated
```
*Take note of the URL generated for `movie-sync-service` (e.g. `https://movie-sync-service-xxxx-de.a.run.app`).*

## 2. Configure IAM & Cloud Scheduler (OIDC Verification)

Follow these steps to authorize Cloud Scheduler to securely trigger the sync route daily at 2:00 AM.

### Step 1: Create a Dedicated Service Account
Create an IAM service account that will represent the Cloud Scheduler trigger:
```bash
gcloud iam service-accounts create scheduler-run-invoker \
  --display-name="Cloud Scheduler Run Invoker"
```

### Step 2: Grant Invoker Role to the Service Account
Authorize this service account to call the private `movie-sync-service` Cloud Run instance:
```bash
gcloud run services add-iam-policy-binding movie-sync-service \
  --member="serviceAccount:scheduler-run-invoker@<PROJECT_ID>.iam.gserviceaccount.com" \
  --role="roles/run.invoker" \
  --region=asia-east1
```

### Step 3: Create the Cloud Scheduler Cron Job (Daily 2:00 AM)
Create the scheduler trigger to send a POST request to the `/api/v1/movies/sync` endpoint:
```bash
gcloud scheduler jobs create http movie-sync-daily-job \
  --schedule="0 2 * * *" \
  --uri="https://movie-sync-service-xxxx-de.a.run.app/api/v1/movies/sync" \
  --http-method=POST \
  --oidc-service-account-email="scheduler-run-invoker@<PROJECT_ID>.iam.gserviceaccount.com" \
  --oidc-token-audience="https://movie-sync-service-xxxx-de.a.run.app/api/v1/movies/sync" \
  --time-zone="Asia/Taipei"
```

*Trigger Configuration Details:*
- `--schedule="0 2 * * *"`: Cron pattern for **daily at 02:00 AM**.
- `--http-method=POST`: Triggers the POST endpoint.
- `--oidc-service-account-email`: Specifies the identity to generate the OIDC token.
- `--oidc-token-audience`: Audience matches the sync service endpoint URL. Cloud Run will automatically reject the token if the audience does not match.
- `--time-zone="Asia/Taipei"`: Forces schedule alignment with Taipei time.
